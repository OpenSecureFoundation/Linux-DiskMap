# Conception et mise en œuvre de DiskMap, un visualiseur de l’occupation et de la fragmentation du disque: cas d’Ubuntu

# Objectifs:

1. Cartographier les blocs utilisés par les fichiers ;
2. Estimer la fragmentation ;
3. Visualiser la répartition de l’espace libre ;
4. et produire un rapport sur l’état global du disque.
  
# Fonctionnalités attendues
1. Lire une une partition et en extraire : sa taille totale, l’espace utilisé et l’espace libre ;le nombre de blocs et leur taille.
2. Identifier les blocs alloués à chaque fichier et calculer : le degré de fragmentation,la répartition des blocs par fichier.
3. Générer une dashboard illustrant l’utilisation des blocs : vert = libre, bleu = contigu, rouge = fragmenté
4. Fournir un rapport global : pourcentage de fragmentation, fichiers les plus fragmentés, espace gaspillé.
